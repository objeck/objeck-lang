Objeck Text Editor Support

 

The following outlines how to enable code syntax highlighting for various editors. Preferred editor is Visual Studio Code, which is cross platform, and provides syntax highlighting and LSP support.

 

Visual Studio Code (Windows/Linux/macOS)

Support for syntax highlighting and compiling code.

Windows: Copy all the VS Code files to the directory “%USERPROFILE%\.vscode\extensions\objeck”.

Restart the application

 

Linux and macOS: Copy all the VS Code files to the directory “~/.vscode/extensions/objeck”. Restart the application

 

LSP Support

Please refer to the instructions on the Objeck LSP project page.

 

Sublime (Windows/Linux/macOS)

Support for syntax highlighting and compiling code.

Windows: Copy the objeck.sublime-syntax, objeck.tmPreferences and objeck.sublime-build files to “%USERPROFILE%\AppData\Roaming\Sublime Text\Packages\Objeck” directory

Restart the application

 

Linux: Copy the objeck.sublime-syntax, objeck.tmPreferences and objeck.sublime-build files to the “~/.config/sublime-text/Packages/Objeck” directory

Restart the application

 

macOS: Copy the objeck.sublime-syntax, objeck.tmPreferences and objeck.sublime-build files to the “~/Library/Application Support/Sublime Text/Packages/Objeck” directory

Restart the application.

 

Ensure the “build” command path is absolute or that the Objeck bin directory is in your environment path.

 

For Windows:

{

     "env": {

           "OBJECK_LIB_PATH": "<install_path>/lib",

     },

     "cmd": ["<install_path>/bin/obc", "-src", "$file", "-lib", "collect.obl,gen_collect.obl,net.obl,rss.obl,misc.obl,regex.obl,csv.obl,xml.obl,json.obl,cipher.obl,odbc.obl,query.obl,sdl2.obl,sdl_game.obl", "-dest", "$file_base_name.obe"],

     "selector": "source.objeck-obs",

     "file_regex": "^(..[^:]*):([0-9]+):?([0-9]+)?:? (.*)$"

}

 

For Linux and macOS:

{

"env": {

     "OBJECK_LIB_PATH": "<install_path>/lib",

     "PATH": "<install_path>/bin"

},

     "cmd": ["obc", "-src", "$file", "-lib", "collect.obl,gen_collect.obl,net.obl,rss.obl,misc.obl,regex.obl,csv.obl,xml.obl,json.obl,cipher.obl,odbc.obl,query.obl,sdl2.obl,sdl_game.obl", "-dest", "$file_base_name.obe"],

     "selector": "source.objeck-obs",

     "file_regex": "^(..[^:]*):([0-9]+):?([0-9]+)?:? (.*)$"

}

 

LSP Support

 

Install the Sublime LSP package and the Objeck LSP server.

 

Open Preferences > Package Settings > LSP > Settings and add the "objeck-lsp" client configuration to the "clients":

 

{

     "clients": {

          "objeck": {

               "enabled": false,

               "command": [

                    "<objeck_path>/bin/obr.exe",

                    "<objeck_server_path>/objeck_lsp.obe",

                    "<objeck_server_path>/objk_apis.json",

                    "6013",

                    "debug"

               ],

               "env": {

                         "OBJECK_LIB_PATH": "<objeck_path>/lib"

               },

               "selector": "source.objeck-obs",

               "tcp_port": 6013

          }

     }

}

 

CudaText (Windows/Linux/macOS)

Plugin support is available. Go to “Plugins / Addons Manager / Install" and select Objeck

 

Notepad++ (Windows)

Support for syntax highlighting

Open application click “Language” ”Define your language”

Select “Import” and navigate to the objeck.xml file

Restart the application

 

gEdit (Linux)

Support for syntax highlighting

Copy objeck.lang to the “/usr/share/gtksourceview-3.0/language-specs/”

Restart the application
